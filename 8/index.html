Question:

Here is a recursive C function that returns the nth number (zero-indexed) in the Fibonacci series. 

int fib_1 (int n)
{
    if (0 == n)
        return 0;
    if (1 == n)
        return 1;

    return fib_1 (n - 1) + fib_1 (n - 2);
}

Calling fib_1 with n=42 takes about 2.5 seconds on a test machine with a Core i5 CPU. Now consider the following implementation: 

static int fn (int n, int a, int b)
{
    if (0 == n)
        return a;
    if (1 == n)
        return b;

    return fn (n - 1, b, a + b);
}

int fib_2 (int n)
{
    return fn (n, 0, 1);
}

fib_2 returns the same result as fib_1, but finishes in less than a millisecond when executed on the same piece of hardware. Why is fib_2 significantly more efficient than fib_1?
